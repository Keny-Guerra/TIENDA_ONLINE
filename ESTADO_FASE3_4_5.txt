📦 FASE 4: SISTEMA DE PEDIDOS
================================================================================

✅ COMPLETADO:
  • Endpoint POST /api/pedidos - Crea nuevo pedido
  • Endpoint GET /api/pedidos - Obtiene todos los pedidos
  • Endpoint GET /api/pedidos/:id - Obtiene pedido específico
  • Endpoint PUT /api/pedidos/:id/estado - Actualiza estado
  • Función procesarPago() - Crea pedido desde pago
  • Función verMisPedidos() - Muestra historial de usuario
  • Cálculo automático de fecha entrega (+5 días)
  • Código de pedido único formato GOL-{id}

🔄 FLUJO DE COMPRA:
  1. Usuario agrega productos al carrito
  2. Click en "Procesar Pago"
  3. Modal valida: carrito no vacío + usuario autenticado
  4. Muestra resumen de compra
  5. Usuario selecciona método de pago
  6. Al confirmar: POST /api/pedidos
  7. Se crea registro con:
     - Datos cliente (nombre, email, teléfono)
     - Total calculado
     - Estado: "confirmado"
     - Fecha entrega estimada (fecha actual + 5 días)
  8. Se retorna codigoPedido (GOL-X)
  9. Carrito se limpia automáticamente

📊 TABLA PEDIDOS:
  - id: INT (PK)
  - cliente_nombre: VARCHAR(255)
  - cliente_email: VARCHAR(255)
  - cliente_telefono: VARCHAR(20)
  - proveedor_id: INT (FK)
  - total: DECIMAL(10,2)
  - estado: VARCHAR(50) DEFAULT 'pendiente'
  - fecha_entrega_estimada: DATE
  - created_at: TIMESTAMP
